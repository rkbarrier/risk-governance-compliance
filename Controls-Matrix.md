# Enterprise Controls Matrix

## Executive Overview

An effective controls framework helps organizations reduce operational risk, strengthen governance, and support regulatory compliance. This Controls Matrix demonstrates how preventive, detective, and corrective controls can be aligned to key business processes within an enterprise payment environment.

> **Portfolio Note:** This document is an original example created for professional portfolio purposes. It is inspired by enterprise banking governance practices and does not contain confidential or proprietary employer information.

---

# Purpose

The purpose of this Controls Matrix is to:

* Document key business and technology controls.
* Define ownership and accountability.
* Support governance and audit readiness.
* Reduce operational and compliance risk.
* Promote consistent control execution.

# Control Categories

### Preventive Controls

Designed to stop issues before they occur.

Examples:

* User authentication
* Role-based access
* Approval workflows
* Automated validation rules

### Detective Controls

Designed to identify issues after they occur.

Examples:

* Exception reporting
* Monitoring dashboards
* Audit logging
* Reconciliation activities

### Corrective Controls

Designed to restore normal operations and prevent recurrence.

Examples:

* Disaster recovery procedures
* Root cause analysis
* Corrective action plans
* Lessons learned reviews
